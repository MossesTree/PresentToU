# PresentToU 디자인 기준

PresentToU는 선물을 준비하는 따뜻한 설렘과 손편지의 정서를 전달하는 서비스다. 화면은 빈티지 문구류와 손그림 선물 일러스트를 바탕으로 하며, 모든 기능 화면은 같은 색·글꼴·간격·움직임 규칙을 따른다.

## 1. 디자인 방향

- 핵심 인상: 따뜻함, 기대감, 정성, 작은 놀라움
- 시각 언어: 크림색 종이, 파스텔 핑크와 블루, 금빛 포인트, 손그림 윤곽선
- 화면 분위기: 장식적이지만 정보가 묻히지 않는 차분한 빈티지 문구점
- 우선순위: 읽기 쉬운 본문과 버튼 > 장식 요소 > 애니메이션

## 2. 색상 팔레트

### 기본 색상

| 역할 | 색상 | 사용 기준 |
| --- | --- | --- |
| 화면 배경 | `#FBF3E6` | 전체 배경, 큰 여백 |
| 종이 표면 | `#FFFCF5` | 편지지, 카드, 입력 영역 |
| 연한 핑크 | `#F7DEE2` | 봉투, 보조 배경, 부드러운 강조 |
| 핑크 | `#E7AFB8` | 주요 장식, 활성 상태 |
| 진한 핑크 | `#D48A97` | 강조 글자, 링크, 작은 장식 |
| 연한 블루 | `#D8E4F0` | 보조 장식, 차분한 강조 |
| 블루 | `#A6C1DC` | 아이콘, 보조 버튼, 선물 장식 |
| 진한 블루 | `#6E90B4` | 블루 계열 강조 글자 |
| 잉크 | `#4B4038` | 제목, 본문, 윤곽선 |
| 보조 잉크 | `#83766A` | 설명, 비활성 정보 |
| 금빛 | `#C7A56A` | 배지, 별, 리본, 별점 |
| 선 | `#D9BFC0` | 테두리, 구분선 |

### 사용 원칙

- 본문은 `#4B4038`, 설명은 `#83766A`를 사용한다.
- 밝은 배경 위에 보조 잉크만 단독으로 사용하지 않는다. 작은 글자는 잉크 색을 우선한다.
- 주요 행동 버튼은 진한 잉크 또는 진한 핑크를 배경으로 사용하고, 글자는 종이색으로 표시한다.
- 핑크와 블루는 같은 영역에서 강하게 경쟁시키지 않고, 한 화면의 주 강조색을 하나로 정한다.
- 반투명 배경을 사용할 때는 종이색을 기준으로 `rgba(255, 252, 245, .9)` 이상을 사용해 가독성을 확보한다.

## 3. 글꼴과 타이포그래피

```css
--font-sans: "Noto Sans KR", "Pretendard", system-ui, sans-serif;
--font-serif: "Noto Serif KR", "Noto Sans KR", serif;
```

- 제목과 감성적인 짧은 문구는 `Noto Serif KR`를 사용한다.
- 본문, 버튼, 입력 안내, 메뉴는 `Noto Sans KR`를 사용한다.
- 제목은 너무 검거나 촘촘하게 만들지 않고, 줄 간격을 `1.25~1.5`로 유지한다.
- 본문은 최소 `14px`, 주요 안내 문구는 최소 `15px`를 권장한다.
- 장식용 영문 라벨은 대문자와 넓은 자간을 사용할 수 있지만, 정보 전달은 한국어로 함께 제공한다.

## 4. 모서리와 표면

- 일반 카드: `border-radius: 16px~22px`
- 주요 안내 배지와 버튼: `border-radius: 999px`
- 큰 소개 영역: `border-radius: 24px~28px`
- 테두리는 `1px solid #D9BFC0` 계열로 가볍게 표현한다.
- 그림자는 넓고 흐리게 사용한다. 검은 그림자보다 갈색 계열의 낮은 투명도를 사용한다.

```css
box-shadow: 0 5px 14px rgba(120, 90, 70, .16);
```

- 카드와 종이는 완전한 흰색보다 `#FFFCF5`를 사용한다.
- 손그림 느낌이 필요한 일러스트에는 잉크색 윤곽선과 미세한 흔들림을 사용하되, 텍스트와 입력 UI에는 적용하지 않는다.

## 5. 컴포넌트 기준

### 카드

- 종이색 배경, 얇은 핑크 계열 테두리, 부드러운 갈색 그림자를 사용한다.
- 카드 안쪽 여백은 모바일 `20px`, 데스크톱 `24~28px`를 기준으로 한다.
- 장식 아이콘은 카드 내용보다 먼저 보이되, 본문보다 크지 않게 한다.

### 버튼

- 주요 버튼은 한 화면에 하나를 우선 노출한다.
- 버튼 안의 문장은 짧은 동사형으로 작성한다. 예: `선물 추천 시작하기`.
- 기본 높이는 `44px` 이상으로 유지해 모바일에서도 누르기 쉽게 한다.
- 호버 시에는 색을 과하게 바꾸기보다 `translateY(-2px)` 정도의 작은 움직임을 사용한다.

### 배지와 안내 문구

- 선물, 분석 상태, 추천 근거처럼 짧은 정보를 표현할 때 사용한다.
- 선물 일러스트 위에 문구가 놓이면 종이색 반투명 배경과 테두리를 함께 사용한다.
- 안내 문구는 낮은 대비의 회색만으로 표시하지 않는다.

### 서비스 소개 탭

- 첫 화면과 같은 크림색 배경과 파스텔 장식색을 사용한다.
- 상단 소개 영역에는 짧은 감성 문구, 서비스가 하는 일, 주요 행동 버튼을 순서대로 배치한다.
- 기능 설명은 `대화 업로드 → AI 취향 분석 → 맞춤 선물 추천`의 흐름이 한눈에 보이게 구성한다.
- 기능 카드는 같은 표면·테두리·모서리 규칙을 공유하고, 활성 기능만 핑크 또는 블루로 구분한다.
- 후기와 추천 근거는 종이 카드 형태로 표현하며, 인용 부호와 별점에는 금빛을 사용한다.
- 마지막 행동 영역은 진한 잉크색 또는 진한 핑크색으로 대비를 주되, 다른 화면의 주요 버튼과 같은 문장 스타일을 사용한다.

## 6. 애니메이션

- 애니메이션은 사용자의 시선을 안내할 때만 사용한다.
- 봉투와 편지지는 `0.6~1.0초`의 부드러운 전환을 사용한다.
- 반복 애니메이션은 안내 문구처럼 주의를 환기해야 하는 요소에만 적용한다.
- 장식 요소가 본문과 버튼의 가독성을 방해하지 않도록 장식 레이어는 콘텐츠보다 낮은 우선순위로 둔다.
- `prefers-reduced-motion: reduce` 환경에서는 반복 애니메이션과 큰 이동을 제거한다.

## 7. 반응형 기준

- 모바일 기본 여백은 좌우 `20~24px`로 둔다.
- 데스크톱 콘텐츠 최대 너비는 `1120~1152px`를 기준으로 한다.
- 카드 그리드는 모바일에서 1열, 넓은 화면에서 2~3열로 전환한다.
- 장식 일러스트는 모바일에서 크기와 개수를 줄이고, 핵심 문구 주변의 여백을 우선 확보한다.
- 버튼과 링크는 터치 영역이 최소 `44px`가 되도록 한다.

## 8. 구현 규칙

- 기능별 CSS는 해당 기능 폴더의 HTML 안에서 역할별 구획을 유지한다.
- 색상과 글꼴은 가능한 한 CSS 변수로 관리해 화면 간 값을 직접 복사하지 않는다.
- 계산·상태 변경·DOM 표시 로직을 분리한다.
- 시각적 장식 때문에 본문 대비가 낮아지면 장식을 먼저 줄이고 텍스트를 우선한다.
- 새 화면을 만들 때는 이 문서의 색상, 표면, 간격, 버튼, 애니메이션 규칙을 먼저 적용한 후 개별 장식을 추가한다.
