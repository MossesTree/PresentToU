## 3. 평가 하네스 결함 발견·수정 (이번 평가의 부산물)

sol 심사 1차 결과는 **6/24 (25%)** 였고, 실패 18건이 전부 같은 사유("입력에 없는 사실을 근거로 지어냄")였다.
원인 분석 결과 **심사 프롬프트에 입력 '요약문'만 있고 실제 대화 원문이 없어**, 심사자가 인용의 실재 여부를 검증할 수 없는 구조였다:

- 관대한 gpt-4o-mini는 이 상태에서 무작위로 오판(이전 실행들의 오판 3~4건 원인)
- 엄격한 gpt-5.6-sol은 "검증 불가 = 환각"으로 일괄 판정 → 25%

**수정**(커밋 `16d41de`): 심사 프롬프트에 `[실제 입력 데이터]`(대화 원문·설정)를 전달하고, 원문과 대조해 인용·환각을 판정하도록 규칙 명시. 수정 후 재심사(재생성 없음)에서 **83%**로 정상화됐고, 남은 실패는 원문 문구 수준의 정밀한 판정으로 확인됐다.

> 교훈: LLM-as-a-judge는 심사에 필요한 원본 데이터를 전부 줘야 한다. 심사 모델이 엄격할수록 하네스 결함이 극단적으로 드러난다.

## 4. 남은 실패 4건 — "근거 미세 각색" 패턴

sol은 대놓고 지어내지는 않지만, **원문을 근거로 옮기며 미세하게 각색**하는 패턴이 확인됐다:

| 케이스 | 원문 | 출력 근거 | 문제 |
|--------|------|-----------|------|
| F1-Q2 | "저녁엔 러닝 시작했어 5km씩 뛰어" | "저녁마다 5km 러닝" | 빈도(매일) 추가 |
| F3-Q2 | "도구 욕심도 조금씩 생기고 ㅋㅋ" | 「도구 욕심도 조금씩 생겼어」(직접 인용 표기) | 인용문 변형 |
| F4-Q4 | '나'가 "산책하기 딱이야"라고 말함 | 받는 사람이 산책을 즐긴다고 근거화 | **화자 오귀속** |
| F3-Q3 | — | 203,509원(배송·관부가세 별도) | 최종가 불확실로 예산 검증 불가 |

**개선 방향**: 생성 프롬프트에 ① "직접 인용은 원문 그대로, 한 글자도 바꾸지 마라" ② "발언의 화자를 확인하고 받는 사람의 발언만 취향 근거로 사용하라" ③ "해외 배송 상품은 최종 결제가 기준으로 예산을 판단하라" 규칙 추가.

## 5. 모델 비교 결론

| 관점 | gpt-4o-mini | gpt-5.6-sol |
|------|-------------|-------------|
| 민감 내용(F5) | 중립 언급 노출 (4/5 실패) | **전 문항 통과, 언급 0회** |
| 근거 정직성 | 출처 오표기, 근거 없는 단정 | 우수하나 미세 각색·화자 오귀속 잔존 |
| 예산 준수 | 하한 미달 사례 | 범위 준수 양호 (해외가 표기만 이슈) |
| 응답 속도 | ~40초/케이스 | ~80초/케이스 (60초 UX 목표에 불리) |
| 심사자로서 | 오판 잦음 (산수·인용 대조 실패) | 정밀함 — 원문 문구 수준 대조 |

**권장**: 품질 기준으로는 sol 전환이 타당하다. 단 ① 응답 시간이 2배라 60초 목표(FR-008)에 부담 — 대기 UX 보강 또는 시간 예산 가드 병행 권장, ② 4절의 프롬프트 개선 3건 적용 후 재평가 권장.

> 비교 주의: 4o-mini 실행(46%)은 하네스 수정 전 심사라 수치의 직접 비교는 제한적이다. 다만 두 실행 모두 출력 원본을 수동 대조해 분류했으므로 결함 패턴 비교(F5 해결 등)는 유효하다.
