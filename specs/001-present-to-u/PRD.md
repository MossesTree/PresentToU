# presen2U — PRD (v2, 구현 반영본)

> v1(질문 몇 개 → 규칙 기반 큐레이션)에서 **대화 데이터 기반 AI 추천**으로 제품이 진화한 것을 반영해 갱신한 문서.
> 필드 구성은 원본 `idea` 양식(문제·타깃·유저스토리·핵심기능·비목표·성공기준)을 따른다.
> 점검 체크리스트: [[prd-quality]] (`checklists/prd-quality.md`) · 기능 명세: `spec.md` · 품질 기준: `golden-set.md`

## 문제

친밀도가 낮은 상대에게 선물해야 할 때, 상대의 취향 정보가 부족한 상태에서 너무 많은 선택지를 비교하느라 **제때 하나를 결정하지 못한다.** 취향 단서는 사실 그 사람과 나눈 대화 속에 이미 있지만, 사람이 긴 대화를 뒤져 단서를 뽑아내기는 어렵다.

## 타깃

친밀도가 낮은 관계(직장 동료·지인·모임 등)에게 선물을 골라야 하는 20~30대. 상대와의 카카오톡 대화나 통화 녹음을 가지고 있는 사용자.

## 유저스토리

취향을 잘 모르는 상대에게 선물을 준비하는 사람으로서, 나는 **그 사람과 나눈 대화를 올리면 실제 대화 근거가 붙은 선물 후보를 빠르게 받아 결정하고 싶다.** 왜냐하면 수많은 상품을 비교하며 시간을 낭비하고 싶지 않고, "왜 이 선물인지" 확신도 얻고 싶기 때문이다.

## 핵심기능

**대화 데이터를 올리면 AI가 취향 신호를 추출해 실제 구매 가능한 TOP 3 선물 카드와 전할 편지까지 만들어 준다.** (단일 flow)

```
[봉투 인트로] → [입력: 받는 사람·성별·관계·예산(또는 무관)·선물 목적·추가 정보
               + 카카오톡 대화 내보내기(.txt)·통화 음성(4MB 이하) 업로드]
            → [AI 분석: 6단계 진행 표시 + 대기 미니게임]
            → [결과: TOP 3 카드(플립 상세) + 피해야 할 선물 1개 + 맞춤 편지(편집·이미지 저장)]
```

- **카드 구성**: 앞면 = 선물 이름·이모지·추천 이유 2문장·적합도 점수(fitScore)·예상 가격대 / 뒷면(커서 오버 플립) = 상세 설명·대화 근거(직접 인용)·취향 신호·구매 링크.
- **AI 파이프라인**: 긴 대화 요약 → 취향 분석·TOP 3 생성 → 웹검색으로 실제 판매 상품·구매 URL 특정(병렬) → 민감 내용 검수. 단계별 폴백으로 항상 정확히 3개 카드를 보장.
- **wow 지점**: "대화에서 ~라고 말씀하신 점" 같은 **실제 대화 인용 근거**와, 바로 살 수 있는 **실상품 구매 링크**, 그리고 관계·목적에 맞는 톤의 **전할 편지**까지 한 번에.

### 품질 규칙 (제품 제약)

- **예산 존중**: 추천 가격은 입력 예산 범위 내. 예산대에 따라 기대 품질 등급(센스형/실속형/프리미엄형/럭셔리형)을 내부 판단해 추천 급을 맞춘다.
- **근거 정직성**: 대화에 없는 사실을 근거로 지어내지 않는다. 근거가 없으면 명시하고 적합도 점수에 상한(65점)을 둔다.
- **민감 내용 금지**: 성·정치·종교·혐오 표현은 분석·카드·편지 어디에도 노출하지 않는다(생성 금지 규칙 + 표시 전 검수·교체 2중 차단).
- **새 제품·신뢰 판매처만**: 중고·리퍼·전시 상품 금지, 구매 링크는 신뢰 판매처(쿠팡·네이버쇼핑·11번가 등 주요 커머스)만 허용 — 그 외 링크는 제거하고 검색 링크로 대체.
- **개인정보 무저장**: 업로드된 대화·음성은 추천 생성에만 쓰고 저장하지 않는다.

## 비목표

- 회원가입 / 로그인
- 외부 데이터 자동 연동(카카오톡 API 연동, 쇼핑몰 재고·가격 실시간 연동 등) — 대화는 **수동 업로드**
- 실제 구매 및 결제 (전환은 외부 상품 페이지 이동까지)
- 위시리스트 저장, 추천 이력, 개인화 학습 모델

## 성공기준

| 지표 | 정의(측정 시작~종료) | 목표 |
| ------ | --------------------------------------------------- | ------ |
| 결정 속도 | **추천 요청** ~ **TOP 3 카드 첫 노출**까지 경과 시간 | < 60초 (웹검색 지연 시 최대 300초, 대기 UX로 보완) |
| 추천 유효성 | 데모 세션 중 **카드의 외부 상품 링크를 1개 이상 클릭한 세션** 비율 (분모=전체 세션) | > 30% |
| 추천 품질 | 골든셋(24문항 자동 심사, LLM-as-a-judge 루브릭 4점 이상) 통과율 | ≥ 80% (2026-07-15 측정 83%) |
| 표본 | 데모 참가자(세션) 수 | n ≥ 20 |

**용어 정의(모호어 제거)**
- **"실패하지 않을 선물"** = 수령자가 부정적 반응 없이 무난하게 받아들일 수 있는 선물(데모에서는 외부 상품 링크 클릭률로 대리 측정).
- **"대화 근거"** = 업로드된 대화 원문에 실재하는 발언의 직접/간접 인용. 원문에 없는 내용은 근거로 인정하지 않는다.
- **"신뢰 판매처"** = 코드에 정의된 주요 커머스 허용 목록(`api/_logic.js`의 TRUSTED_STORE_DOMAINS). 중고 거래 플랫폼은 무조건 제외.
- 금지어("빠르게·편하게·예쁘게·잘" 등 정성 표현)는 모두 수치·조작적 정의로 대체.

## 간단 설명

- **핵심 한 문장**: 그 사람과 나눈 대화를 올리면, AI가 대화 속 취향 단서를 근거로 바로 살 수 있는 선물 3개와 전할 편지까지 만들어 주는 서비스.
- **비유**: 눈치 빠른 친한 친구에게 "얘랑 나눈 대화 좀 봐봐, 뭐 사주면 좋을까?"라고 묻는 것과 같다 — 친구는 대화에서 "요즘 캠핑에 빠졌다더라"를 짚어내고, 매장 링크까지 찾아주고, 카드에 쓸 문구까지 대신 써 준다.
- **초보자용 설명**: 카카오톡 대화 파일을 올리면 AI가 상대의 관심사를 찾아 선물 3개를 추천한다. 각 추천에는 "대화에서 이렇게 말했기 때문"이라는 근거와 실제 쇼핑몰 구매 링크가 붙고, 선물과 함께 전할 편지도 만들어 준다. 회원가입도 결제도 없고, 올린 대화는 저장되지 않는다.
