# presen2U — PRD (보완본 v1)

> 원본 아이디어를 6대 기준으로 점검·보완한 문서. 필드 구성은 원본 `idea` 양식(문제·타깃·유저스토리·핵심기능·비목표·성공기준)을 따른다.
> 점검 체크리스트: [[prd-quality]] (`checklists/prd-quality.md`)

## 문제

친밀도가 낮은 상대에게 선물해야 할 때, 상대의 취향 정보가 부족한 상태에서 너무 많은 선택지를 비교하느라 **제때 하나를 결정하지 못한다.**

## 타깃

친밀도가 낮은 관계(직장 동료·지인·모임 등)에게 선물을 골라야 하는 20~30대.

## 유저스토리

취향을 잘 모르는 상대에게 선물을 준비하는 사람으로서, 나는 **1분 안에 무난하게 받아들여질 선물 후보를 결정하고 싶다.** 왜냐하면 수많은 상품을 비교하며 시간을 낭비하고 싶지 않기 때문이다.

## 핵심기능

**질문 몇 개에 답하면 1분 안에 TOP 3 선물 카드가 뜬다.** (단일 flow)

```
[시작 버튼] → [3~5개 객관식 질문: 관계 · 예산대 · 성별/연령대 · 취향 키워드]
           → [TOP 3 선물 카드: 이름 · 대표 이미지 · 추천 이유 1줄 · 예상 가격대]
```

- wow 지점: 입력 최소화(탭 몇 번) → 1분 내 결정 가능한 3개로 압축.
- 추천 로직은 규칙 기반/사전 큐레이션 데이터로 충분(정교한 개인화 아님).

## 비목표

- 회원가입 / 로그인
- 외부 데이터 자동 연동(쇼핑몰 재고·가격 실시간 연동 등)
- 실제 구매 및 결제
- 위시리스트 저장, 추천 이력, 개인화 학습

## 성공기준

| 지표     | 정의(측정 시작~종료)                                        | 목표     |
| ------ | --------------------------------------------------- | ------ |
| 결정 속도  | **시작 버튼 클릭** ~ **TOP 3 카드 첫 노출**까지 경과 시간            | < 60초  |
| 추천 유효성 | 데모 세션 중 **TOP 3 카드 중 1개 이상을 클릭한 세션**의 비율 (분모=전체 세션) | > 30%  |
| 표본     | 데모 참가자(세션) 수                                        | n ≥ 20 |

**용어 정의(모호어 제거)**
- **"실패하지 않을 선물"** = 수령자가 부정적 반응 없이 무난하게 받아들일 수 있는 선물(데모에서는 클릭률로 대리 측정).
- **"1분 안에"** = 위 표의 *결정 속도* 정의(버튼 클릭~카드 노출 <60초)를 따른다.
- 금지어("빠르게·편하게·예쁘게·잘" 등 정성 표현)는 모두 수치·조작적 정의로 대체.

## 간단 설명

- **핵심 한 문장**: 취향을 모르는 상대에게 줄 선물을, 질문 몇 개로 1분 안에 TOP 3로 좁혀 결정 부담을 없애는 서비스.
- **비유**: 넷플릭스가 수천 편 중 "오늘 볼 만한 3편"을 골라주듯, 수많은 상품 중 "이 사람에게 무난한 3개"를 골라준다.
- **초보자용 설명**: 무엇을 살지 고민될 때 앱이 몇 가지를 물어보고, 고르기 쉽게 딱 3개만 추천해 준다. 회원가입도 결제도 없이 "추천만" 하는 게 이번 목표다.
