# Quickstart: PresentToU 검증 가이드

**Plan**: [plan.md](./plan.md) | **계약**: [recommend](./contracts/recommend-api.md), [transcribe](./contracts/transcribe-api.md)

## 사전 준비

- 브라우저(데스크톱, hover 가능 환경 권장)
- API 동작 검증 시: Vercel 배포 + 환경변수 `OPENAI_API_KEY`, `GROQ_API_KEY` 설정
  (로컬은 `npx vercel dev`로 서버리스 함수 포함 실행)
- 샘플 데이터: 카카오톡 대화 내보내기 `.txt` 1개, 음성 파일(`.m4a` 등, 4MB 이하) 1개

## 실행

```bash
# 정적 UI만 확인: index.html을 브라우저로 직접 연다
start index.html          # Windows

# API 포함 전체 흐름: Vercel 개발 서버
npx vercel dev
```

## 검증 시나리오

### S1. 핵심 흐름 — 업로드 → 60초 내 TOP 3 (US1, FR-001~004, FR-008)

1. 첫 화면 봉투를 클릭해 열고, 받는 사람 이름을 입력해 계속 진행
2. 성별·관계·예산을 선택하고 카카오톡 `.txt` 파일을 업로드(드래그 앤 드롭도 확인)
3. 추천 시작 → 6단계 진행 표시가 순서대로 바뀌는지 확인
4. **기대**: 추천 요청 시작부터 60초 미만에 선물 카드가 **정확히 3개** 노출(SC-001, SC-004)
5. 음성 파일을 함께 올린 경우: "음성 전사" 단계가 표시되고, 결과 근거에 통화 내용 신호가 반영되는지 확인(FR-002, US1/AC3)

### S2. 카드 플립 → 외부 상품 링크 (US2, FR-005~007)

1. TOP 3 카드 앞면에 이름·이모지·추천 이유·예상 가격대가 있는지 확인(FR-005)
2. 카드에 커서를 올리면 뒤집혀 상세(설명·대화 근거·취향 배지·링크)가 보이는지 확인(FR-006)
3. 외부 상품 링크 클릭 → 새 탭으로 판매 페이지(또는 네이버 쇼핑 검색)가 열리는지 확인(FR-007)

### S3. 추천 근거 (US3, FR-003)

- 각 카드의 추천 이유가 대화 신호(직접/간접 인용)와 연결된 문장인지 확인
- 대화에 근거가 없는 항목은 "단서를 확인하지 못했습니다"로 표기되는지 확인(환각 금지)

### S4. 오류·데이터 부족 처리 (FR-011, FR-012, SC-005)

| 입력 | 기대 동작 |
|------|-----------|
| 지원하지 않는 형식(.pdf 등) | 업로드 거부 + 원인·다음 조치 안내 |
| 4MB 초과 음성 파일 | 거부 + "4MB 이하" 안내 |
| 빈/아주 짧은 대화 파일 | 추천은 진행하되 근거 부족 안내, 빈 화면 없음 |
| 이름·파일 모두 없이 추천 시도 | 추천 버튼 비활성 또는 안내 표시 |
| 서버 오류/타임아웃 | 재시도 안내 표시, 진행 화면에서 멈추지 않음 |

### S5. 개인정보 (FR-013)

- 개발자 도구 → 네트워크/스토리지: 대화 원문이 localStorage 등 영구 저장소에 남지 않는지 확인
- 서버 로그(요청 ID 기준)에 대화 원문이 아닌 메타데이터(길이·플래그)만 남는지 확인

### S6. 공통 품질 게이트 (AGENTS.md 체크리스트)

- 브라우저 콘솔 에러 0건
- `prefers-reduced-motion` 설정 시 반복 애니메이션 감소 확인
- 모바일 폭(375px)에서 카드 1열 전환·버튼 터치 영역 확인
