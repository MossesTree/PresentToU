# Feature Specification: presen2U — 대화 데이터 기반 선물 TOP 3 추천

**Feature Branch**: `001-presen2u`

**Created**: 2026-07-13

**Status**: Draft

**Input**: User description: "친밀도가 낮은 상대에게 줄 선물을, 사용자가 업로드한 대화 데이터(카카오톡 대화 + 통화 음성 녹음) 기반으로 1분 안에 TOP 3 추천하는 서비스. 회원가입·외부 데이터 자동 연동·결제는 비목표."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 대화 데이터 업로드로 1분 안에 TOP 3 받기 (Priority: P1)

취향을 잘 모르는 상대에게 선물을 준비하는 사용자가, 그 상대와 나눈 카카오톡 대화 내보내기 파일과(또는) 통화 음성 녹음 파일을 업로드하면, 서비스가 대화에서 취향 신호를 추출해 실패하지 않을 선물 후보 TOP 3를 1분 안에 제시한다.

**Why this priority**: 이 흐름 하나만으로도 "수많은 상품 비교로 시간 낭비" 문제를 해결하는 핵심 가치가 완성된다. 나머지 기능이 없어도 이 슬라이스만으로 데모가 성립하는 MVP다.

**Independent Test**: 준비된 샘플 대화/음성 파일을 업로드한 뒤, 60초 이내에 정확히 3개의 선물 카드가 노출되는지로 독립 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 사용자가 수령자와의 카카오톡 대화 파일을 가지고 있고, **When** 해당 파일을 업로드하면, **Then** 서비스는 60초 미만에 정확히 3개의 선물 후보 카드를 제시한다.
2. **Given** 사용자가 통화 음성 녹음 파일을 업로드하면, **When** 시스템이 이를 텍스트로 변환·분석하면, **Then** 음성 속 취향 신호가 추천 근거에 반영된다.
3. **Given** 대화 파일과 음성 파일을 함께 업로드하면, **When** 추천이 생성되면, **Then** 두 소스의 신호가 통합되어 TOP 3에 반영된다.

---

### User Story 2 - 카드 플립으로 상세 확인 후 외부 상품 페이지로 이동 (Priority: P2)

사용자가 추천 카드에 커서를 올리면 카드가 뒤집혀(flip) 상세 정보가 나타나고, 상세 정보 안의 외부 상품 페이지 링크를 직접 클릭해 구매를 검토할 수 있다.

**Why this priority**: 추천을 "결정"으로 연결하는 전환 지점이다. 성공기준의 클릭률(외부 상품 링크 클릭)을 만들어내는 핵심 상호작용이지만, P1 추천이 먼저 존재해야 의미가 있다.

**Independent Test**: TOP 3가 노출된 상태에서 한 카드에 커서를 올려 플립·상세 표시가 되고, 상세 안의 외부 링크가 새 페이지로 이동하는지로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** TOP 3 카드가 노출된 상태에서, **When** 사용자가 한 카드에 커서를 올리면, **Then** 카드가 뒤집혀 추천 이유·예상 가격대·외부 상품 링크가 담긴 상세 면이 보인다.
2. **Given** 카드가 뒤집혀 상세가 보이는 상태에서, **When** 사용자가 외부 상품 링크를 클릭하면, **Then** 해당 상품의 외부 페이지로 이동한다(서비스는 결제를 제공하지 않는다).

---

### User Story 3 - 추천 근거 확인으로 신뢰 형성 (Priority: P3)

각 카드에 "왜 이 선물인가"를 대화 신호와 연결한 한 줄 근거가 표시되어, 사용자가 추천을 믿고 빠르게 결정할 수 있다.

**Why this priority**: 근거는 클릭·결정 전환을 높이지만, 추천(P1)과 상호작용(P2)이 갖춰진 뒤 가치가 커지는 보조 요소다.

**Independent Test**: 각 카드에 대화에서 추출된 신호에 근거한 추천 이유가 한 줄로 표시되는지로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** TOP 3가 생성되면, **When** 사용자가 각 카드를 보면, **Then** 카드마다 대화 신호에 기반한 추천 이유 한 줄이 표시된다.

---

### Edge Cases

- 업로드된 대화/음성이 너무 짧거나 취향 신호를 찾을 수 없을 때 시스템은 어떻게 안내하는가?
- 지원하지 않는 파일 형식이거나 파일이 손상/빈 상태일 때 어떻게 처리하는가?
- 음성 인식(텍스트 변환)이 실패하거나 정확도가 낮을 때의 대체 흐름은?
- 대화에 예산·관계 등 핵심 신호가 없을 때 TOP 3를 어떻게 구성하는가?
- 업로드 데이터에 민감한 개인정보가 포함될 때의 취급 방식은?
- 60초 안에 추천을 완성하지 못할 때 사용자에게 무엇을 보여주는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 사용자가 카카오톡 대화 내보내기 파일과 통화 음성 녹음 파일을 업로드할 수 있게 해야 한다(둘 중 하나 또는 둘 다).
- **FR-002**: 시스템은 업로드된 통화 음성 녹음을 텍스트로 변환한 뒤 대화 데이터와 함께 분석해야 한다.
- **FR-003**: 시스템은 대화·음성 데이터에서 수령자의 취향 신호(관심사·선호 카테고리·예산 힌트·관계 맥락 등)를 추출해야 한다.
- **FR-004**: 시스템은 추출된 신호를 바탕으로 정확히 3개의 선물 후보(TOP 3)를 제시해야 한다.
- **FR-005**: 각 추천 카드는 앞면에 선물 이름, 대표 이미지, 추천 이유 한 줄, 예상 가격대를 표시해야 한다.
- **FR-006**: 사용자가 카드에 커서를 올리면 카드가 뒤집혀(flip) 상세 정보 면을 표시해야 한다.
- **FR-007**: 카드 상세 면에는 외부 상품 페이지로 연결되는 링크가 포함되어야 하며, 사용자가 직접 클릭해 이동한다.
- **FR-008**: 시스템은 서비스 사용 시작(업로드 완료 후 추천 요청)부터 TOP 3 첫 노출까지 60초 미만에 완료해야 한다.
- **FR-009**: 시스템은 회원가입·로그인 없이 사용 가능해야 한다.
- **FR-010**: 시스템은 결제 기능을 제공하지 않는다(전환은 외부 상품 페이지 이동까지만).
- **FR-011**: 취향 신호를 추출할 수 없을 정도로 데이터가 부족하면 시스템은 사용자에게 명확한 안내(보완 방법 포함)를 제공해야 한다.
- **FR-012**: 지원하지 않는 형식·손상·빈 파일 업로드 시 시스템은 오류 원인과 다음 조치를 사용자에게 안내해야 한다.
- **FR-013**: 시스템은 업로드된 대화·음성 데이터를 추천 생성에만 사용하고, 데모 세션 처리 후 영구 저장하지 않아야 한다.

### Key Entities *(include if feature involves data)*

- **업로드 데이터**: 수령자 취향 신호의 원천. 카카오톡 대화 내보내기(텍스트)와 통화 음성 녹음(오디오, 텍스트로 변환됨) 두 유형.
- **취향 신호**: 데이터에서 추출된 분석 결과. 관심사·선호 카테고리·예산 힌트·관계 맥락 등을 포함.
- **선물 추천 카드**: 하나의 후보. 이름, 대표 이미지, 추천 이유 한 줄, 예상 가격대, 상세 면의 외부 상품 링크 속성을 가진다.
- **추천 세션**: 업로드 → TOP 3 노출 → 카드 링크 클릭으로 이어지는 1회 사용 흐름. 성공 지표(결정 속도·클릭률)의 측정 단위.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 추천 요청 시작부터 TOP 3 첫 노출까지 60초 미만에 도달한다.
- **SC-002**: 데모 세션 중 TOP 3 카드의 외부 상품 링크를 1개 이상 클릭한 세션 비율이 30%를 초과한다(분모=전체 데모 세션).
- **SC-003**: 위 지표는 표본 n≥20 세션에서 측정한다.
- **SC-004**: 유효한 입력에 대해 시스템은 항상 정확히 3개의 후보를 제시한다.
- **SC-005**: 데이터 부족·오류 입력에 대해 사용자는 다음 조치를 안내받아, 빈 화면 없이 흐름을 이어갈 수 있다.

## Assumptions

- "대화 데이터 기반"은 사용자가 카카오톡 대화 내보내기와 통화 음성 녹음을 **수동 업로드**하는 방식을 의미한다(외부 서비스 자동 연동은 비목표).
- 추천 로직은 데모 수준(취향 신호 매칭 + 사전 큐레이션된 선물 후보/링크)으로 충분하며, 정교한 개인화 학습 모델은 이번 범위 밖이다.
- 외부 상품 링크는 수동으로 큐레이션된 정적 링크를 사용한다(실시간 재고·가격 연동 아님).
- 성공기준의 "클릭"은 카드 상세 면의 **외부 상품 링크 클릭**으로 정의한다(카드 플립/커서 오버는 클릭으로 세지 않는다).
- 업로드 데이터는 개인정보 최소화를 위해 데모 세션 처리 후 저장하지 않으며, 데모에서는 참가자 본인 또는 모의(mock) 대화·음성 데이터를 사용한다고 가정한다.
- 데모 사용 환경은 커서 오버(hover)가 가능한 데스크톱/포인터 기반을 기본으로 가정한다.
